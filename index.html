SiteMe

An easy to use yet versatile website builder. Aims to be easy enough for children to learn yet have enough functionality to create full websites. Will eventually be able to publish the finished product directly from the app.

Next Steps

Exporting/saving. As pdf/images for designers or HTML, CSS and Javascript for developers/to be able to publish website directly.

Templates. Save even more time by having common structures available.

Gamification. Teaching the use of the software via a game could even encourage children to learn how to make/design websites. Starting from placing basic shapes to eventually being able to build an entire website from a given design document.

Tanx

Multiplayer action game with multiple game types and a 4 stage single player mode. Touchscreen ready.

Challenges

Simulate blocked vision. In certain game modes it didn't make sense for the player to be able to see the whole map. This required me to block out objects that were behind walls/other objects.

• To draw the vision blocking polygon, only the two X points were unknown.

• The first step was to calculate the angle of the player to corner of wall.

    fogAngle = Math.atan2(wallCornerX - (playerCenterX), -(wallCornerY - (playerCenterY)));

• So then the Y position of X is: playerCenterY - O and to get O...

        Math.tan(fogAngle) * playerCenterX;

Latency. After trying to render each player everytime they sent their new location to the server (which totally destroyed all animation) I instead only broadcasted player inputs (e.g. key presses).

This resulted in an inconsistency in player positions E.g. if someone holds down a movement direction and releases 0.5 seconds later, but the other client doesn’t register the release till 0.8 seconds later, it will have moved for an extra 0.3 seconds on the other clients screen.

To make up for this, each client sends their position to the server and every so often each client checks the position of other players on their screen against these sent positions.
